# Component library

## What am I building?
The goal is to have a component library relying on cutting-edge CSS & HTML technology.
Also, having great DX for frontend designers is a priority, too.

## Why am I building this?
There are a lot of solutions out there but using as much CSS & HTML possible reduces
dependencies, increases compatibility + has built-in accessibility.
From the design side, the focus is on having APIs and tokens that are easy to understand & use.
Also, it is fun playing around with theming approaches.

## Storybook
This project is built with Storybook from the start meaning the components create can be easily
inspected and tested.
Have a Storybook enables visual testing, catching regressions and improves visibility of available
components.

## Tailwind
Naming is hard, causes friction and costs time.
This is why I believe in the utility-first approach.
I've collected a lot of experience in having to refactor hard to understand files, so a standard set of classes
is the best approach to me.
Also, it makes CSS files way smaller since you are not adding the same property to a new class over and over.

## NextJS
It is a popular framework meaning a lot of people know how to use it and it has great features that 
can be added easily.
