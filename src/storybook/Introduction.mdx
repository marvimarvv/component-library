## What am I building?

The goal is to have a component library relying on cutting-edge CSS & HTML technology.
Having a great DX for frontend development is a priority as well.

## Why am I building this?

There are a lot of solutions out there but using as much CSS & HTML possible reduces
dependencies, increases compatibility + has built-in accessibility.
From the design side, the focus is on having APIs and tokens that are easy to understand and use.
Also, it is just fun playing around with theming approaches.

## Storybook

This project is built with Storybook from the start meaning the components created can be easily
inspected and tested.
Having a Storybook enables visual testing, catching regressions and improves visibility of available
components.

## Tailwind

Naming is hard, causes friction and costs time.
This is why I believe in the utility-first class name approach.
I've collected a lot of experience having to refactor hard to understand files, so a standard set of classes
is the best approach to me.
It also makes CSS files way smaller since you are not adding the same property to a new class over and over which
can make a big difference in larger projects.

## Theming

The library ships with multiple hand-crafted themes — **Gradient** and **Neon** — each available in
light and dark mode. All themes are implemented via CSS custom properties, so switching is as simple
as swapping a class on the root element.

See the dedicated [**Theming**](?path=/docs/theming--docs) page for the full design token reference including primary color palettes,
contrast colors, background variables, and border radii for each theme.
